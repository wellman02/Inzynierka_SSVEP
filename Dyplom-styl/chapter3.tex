\chapter{Projekt oraz implementacja interfejsu SSVEP}

W niniejszym rozdziale przedstawiono główne założenia projektowe oraz ogólną koncepcję implementacji systemu interfejsu mózg-komputer opartego na zjawisku wzrokowych potencjałów wywołanych stanu ustalonego. Celem projektu jest opracowanie rozwiązania umożliwiającego sterowanie kursorem myszy za pomocą odpowiedzi neuronalnych generowanych w odpowiedzi na migoczące bodźce wzrokowe. Konstrukcja systemu musi zapewniać nie tylko funkcjonalność pozwalającą na wykonywanie podstawowych komend ruchu kursora, ale również powtarzalność wyników. Z tego względu nacisk położono na równowagę pomiędzy prostotą konfiguracji a jakością akwizycji sygnału EEG.

\section{Założenia projektowe}

Zdefiniowano dwie grupy wymagań: funkcjonalne, określające możliwości systemu, oraz pozafunkcjonalne, dotyczące jakości i warunków pracy rozwiązania.

\subsection*{Założenia funkcjonalne systemu} 

\begin{itemize}
    \item System umożliwia sterowanie kursorem w czterech kierunkach oraz generowanie kliknięcia.
    \item Użytkownik wybiera komendę poprzez skupienie wzroku na jednym z pięciu migających bodźców.
    \item System dokonuje analizy sygnału w oknach czasowych o stałej długości.
    \item Interfejs pozwala na konfigurację parametrów migających pól.
\end{itemize}

\subsection*{Założenia pozafunkcjonalne systemu} 

\begin{itemize}
    \item Stabilność działania systemu nie zależy od jakości łącza Bluetooth.
    \item Moduł przetwarzania sygnału musi być odporny na zakłócenia w warunkach laboratoryjnych.
    \item Aplikacja działa w sposób deterministyczny.
    \item System działa w czasie rzeczywistym.
\end{itemize}

Ogólna architektura rozwiązania opiera się na interakcji trzech głównych modułów: modułu stymulującego, modułu rejestracji sygnału oraz modułu przetwarzającego. Schemat ogólny modelu systemu, obrazujący przepływ informacji między użytkownikiem a poszczególnymi blokami, przedstawiono na rysunku \ref{fig:model_systemu}.

\section{Implementacja procesu rejestracji i klasyfikacja}

Ostatni podsystem realizuje rejestrację, przetwarzanie oraz klasyfikację sygnału. Dane pobierane są z trzech elektrod zlokalizowanych w punktach O1, O2, Oz (kora wzrokowa), elektrody odniesienia na wyrostku sutkowatym oraz elektrody uziemiającej na drugim wyrostku.

\subsection*{Przetwarzanie sygnału}

Wstępna obróbka sygnału obejmuje filtrację pasmową w zakresie 0.5--30~Hz, co pozwala wyeliminować składową stałą oraz zakłócenia sieci elektrycznej (50~Hz). Następnie na każdym kanale wykonywana jest szybka transformata Fouriera (FFT), przekształcająca sygnał z dziedziny czasu do dziedziny częstotliwości. Fragment kodu realizujący to przekształcenie przy użyciu biblioteki \texttt{numpy} przedstawiono na listingu \ref{alg:fft}.

\begin{lstlisting}[language=Python, caption=Fragment kodu obliczającego FFT, label=alg:fft]
import numpy as np

def compute_fft(x, fs):
    N = len(x)
    X = np.fft.rfft(x)
    freqs = np.fft.rfftfreq(N, 1/fs)
    return freqs, np.abs(X)
\end{lstlisting}

Na podstawie widma wyznaczany jest estymator gęstości widmowej mocy (PSD). Z obliczonego PSD wyodrębniane są pasma odpowiadające częstotliwościom stymulacji (wraz z ich pierwszymi harmonicznymi), a wartości mocy w tych zakresach są sumowane. Algorytm wyznaczania PSD zaprezentowano na listingu \ref{alg:psd}.

\begin{lstlisting}[language=Python, caption=Fragment algorytmu wyznaczania PSD, label=alg:psd]
import numpy as np

def compute_psd(x, fs):
    N = len(x)
    X = np.fft.rfft(x)
    psd = (1/N) * (np.abs(X) ** 2)
    freqs = np.fft.rfftfreq(N, 1/fs)
    return freqs, psd
\end{lstlisting}

\subsection*{Procedura klasyfikacji}

Klasyfikacja polega na wyznaczeniu stosunku sygnału do szumu (SNR) dla każdej docelowej częstotliwości. Maksymalna wartość PSD jest szukana w paśmie $\pm 0.5$~Hz wokół częstotliwości bodźca. Bodziec, dla którego obliczony wskaźnik SNR jest najwyższy, zostaje uznany za intencję użytkownika. Wynik ten jest następnie wykorzystywany do wykonania akcji (ruch kursora lub kliknięcie). Kluczowy fragment logiki klasyfikatora przedstawia listing \ref{alg:klasyk}.

\begin{lstlisting}[language=Python, caption=Fragment algorytmu klasyfikatora opartego na SNR, label=alg:klasyk]
import numpy as np

def compute_snr(freqs, psd, fb, bw=1.0):
    mask_sig = (freqs >= fb - 0.5) & (freqs <= fb + 0.5)
    mask_noise = (freqs >= fb - bw) & (freqs <= fb + bw) & (~mask_sig)
    sig = np.mean(psd[mask_sig])
    noise = np.mean(psd[mask_noise])
    return sig / noise

def classify_snr(x, fs, freqs_targets):
    freqs, psd = compute_psd(x, fs)
    scores = {f: compute_snr(freqs, psd, f) for f in freqs_targets}
    return max(scores, key=scores.get), scores
\end{lstlisting}
